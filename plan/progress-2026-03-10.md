# Progress Report — 2026-03-10

## Summary

Significant display and behaviour improvements: HH:MM:SS time display, text clearing moved to widget layer, TextSpec for font configuration, weekday display, and startup alert suppression. Major refactoring of FocusReminder for clarity.

## Changes

### Time display now HH:MM:SS
- `_format_current_time()` includes seconds
- Time updates every tick (0.5s) instead of once per minute
- Time display widget width increased to 140px

### Text clearing moved from PicoDisplay to Text widget
- `DisplayPort` gained `draw_rect()` method; `measure_text()` added then removed
- `Text` widget takes explicit `width` parameter and clears its own area before drawing
- Text drawn 1px lower to close gap between cleared rectangle and text
- `PicoDisplay.show_text()` is now pure text rendering

### TextSpec class
- New `TextSpec` with font, colour, thickness, and scale (all with defaults)
- Font validated against: sans, gothic, cursive, serif_italic, serif
- Scale is `float` (not `int`) for Hershey/vector font compatibility
- `DisplayPort.show_text()` takes `TextSpec` instead of separate colour/scale params
- `Text` widget takes optional `text_spec` instead of `colour`

### Colour improvements
- Added `rgb()` method returning `(red, green, blue)` tuple
- PicoDisplay adapter uses `*colour.rgb()` for cleaner pen creation
- Simplified test_colour.py to only test `rgb()` (dataclass fields don't need testing)

### Weekday display
- Replaced "Running..." status with weekday name (Monday–Sunday)
- `ClockPort` gained `weekday()` method (0=Monday, 6=Sunday)
- Implemented in `PicoClock` via `time.localtime()[6]`

### Startup alert suppression
- Alerts already due at power-on are auto-dismissed
- Prevents unwanted buzzing when device is powered on late
- `FocusReminder.__init__` checks current time and dismisses past alerts

### FocusReminder refactoring
- Extracted `AlertState.dismiss(today)` to remove duplication
- Extracted `FocusReminder._alert_off()` to remove triplicated off logic
- Flattened `tick()` to use early returns instead of nested if/elif/else
- `_active_alert()` uses early return from sorted states instead of list + min
- `dismissed_on_day` changed from `int | None` to `int` (0 as sentinel, since day-of-year is 1-365)
- Used `_` for unused tuple elements

### Custom test matchers
- Created `tests/matchers.py` with `rect()` and `rect_list()` matchers
- Widget clearing test uses expressive `rect_list(rect(x=10, width=150, colour=BLACK))`
