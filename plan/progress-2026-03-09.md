# Progress Report — 2026-03-09

## Summary

Added colour support, live clock display, and display text clearing to the focus reminder application.

## Changes

### Colour module (`colour.py`)
- New `Colour` class with `red`, `green`, `blue` fields
- Predefined constants: `WHITE`, `BLACK`, `RED`, `GREEN`, `BLUE`, `YELLOW`
- `Text` widget accepts optional colour (defaults to `WHITE`)
- `DisplayPort.show_text` typed with `Colour` parameter
- Reminder labels displayed in blue, times in red

### ApplicationConfig extraction
- Separated construction parameters into `ApplicationConfig` class
- `Application.__init__` now takes a single `config` argument
- Cleaner separation of configuration from behaviour

### Live clock display
- Current time (HH:MM) displayed on the top line next to "Running..."
- Updates automatically each minute (triggered when seconds == 0)
- `ClockPort.current_time()` now returns `(hour, minute, second)` instead of `(hour, minute)`
- All callers updated: `FocusReminder`, `FakeClock`, `PicoClock`

### Display text clearing
- `PicoDisplay.show_text` now clears a black rectangle before drawing text
- Uses `measure_text` for width; accounts for Hershey font midline alignment (y offset -10px, height 30px)
- Fixes ghosting when text updates in place

## Test coverage

32 tests passing, including:
- Colour class fields and constants
- Text widget colour pass-through and default
- Clock time display on start
- Time update on new minute / no update mid-minute
- All existing focus reminder and walking skeleton tests
