# Progress Report — 2026-04-25

## Summary

Fixed British Summer Time handling. The clock now displays UK local time, switching automatically between GMT and BST on the last Sunday of March and October.

## Changes

### `bst_gmt.py` rewritten

The module existed but was never wired in, and contained two bugs:

- `last_sunday()` checked Sakamoto's day-of-week `== 6`, which in the Sat=0 convention is Friday — so BST appeared to start and end two days early.
- The returned weekday was in Sat=0 convention while the rest of the code (matching `time.struct_time`) expects Mon=0..Sun=6.
- The returned `yearday` was hard-coded to 0, which would break `current_date()` rollover detection if the conversion were used.

Rewritten with:
- Typed helpers (`_sakamoto`, `_weekday_mon0`, `_yearday`, `_is_leap`, `_last_sunday`).
- Sunday detection corrected to `_sakamoto(...) == 1`.
- Weekday converted to Mon=0..Sun=6.
- Yearday computed from year/month/day with leap-year handling.
- Shift across midnight handles end-of-month, end-of-year rollover.

### `PicoClock` wired through `utc_to_uk`

`current_time()`, `current_date()`, and `weekday()` now go through a private `_local()` helper that calls `utc_to_uk(time.localtime())`. The port surface is unchanged.

### `deploy.sh`

Added `bst_gmt.py` to the explicit copy list.
